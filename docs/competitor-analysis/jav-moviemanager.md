# 对标分析:JAV_MovieManager

- **仓库**:https://github.com/4evergaeul/JAV_MovieManager
- **分析日期**:2026-06-17
- **分析维度**:演员维度、库管理、头像/资料源
- **规模**:C# / .NET Core + React + Ant Design,~208 星,SQLite
- **定位**:本地影片库管理器,**自己不刮削——刮削外包给外部 MDCX**,专注浏览/演员/播放列表

## 一、定位

与 JvedioNext 同类思路:**刮削外包**(用社区刮削器 MDCX),本项目只做「库管理 + 演员维度 + 播放集成」。自研刮削路线我们更强,刮削层无可借鉴。**价值集中在演员维度深耕**。

## 二、演员维度(本项目强项)

| 能力 | 来源/做法 |
|---|---|
| **女优头像** | 集成 **gfriends**(GitHub 社区维护的女优头像仓库,按女优名组织,有 FileTree.json 索引) |
| **女优资料** | **minnano-av** 站点:评分、身高、年龄、罩杯、三围 |
| **女优大头照** | DMM(可选) |
| **男优刮削** | `AvdanyuScraper.exe` 专门刮男演员(多数工具忽略) |
| 演员多维筛选 | 按年龄 / 身高 / 罩杯 筛选 |
| 关联查询 | 点影片→演员、点演员→其作品,快速导航 |

## 三、其它功能
- PotPlayer 集成 + **批量加入播放列表**(屏幕内所有演员作品一键入列)。
- SQL 自定义查询(高级用户直接查库)。
- 系统托盘运行;增量/全量扫描;Serilog 日志。
- 局限:不支持无码片、欧美片。

## 四、可借鉴提升点

| # | 提升点 | 它的做法 | 我们现状 | 建议 | 优先级 |
|---|---|---|---|---|---|
| 1 | **gfriends 演员头像库** | 集成 GitHub 社区女优头像仓库(FileTree.json 按名索引) | 演员仅名字 | 集成 gfriends 作头像源,**补充 JavBus 详情页头像**(第四次印证演员头像) | 🟡 中 |
| 2 | **minnano-av 演员资料源** | 评分/身高/年龄/罩杯/三围 | 演员仅名字 | 演员资料维度(与 JavBus/MetaTube 互补) | 🟡 中 |
| 3 | **男优(男演员)刮削** | AvdanyuScraper 专刮男优 | 仅女优 | 可选支持男演员维度 | 🟢 低 |
| 4 | 演员多维筛选 + 关联导航 | 年龄/身高/罩杯筛选;点演员↔作品 | 待对照 | 演员页多维筛选 + 关联跳转(呼应 PornBoss) | 🟢 低 |
| 5 | 外部播放器 + 批量播放列表 | PotPlayer 集成,批量入列 | 待对照 | 外部播放器/播放列表(使用侧) | 🟢 低 |

## 五、关键结论

1. **刮削外包(MDCX)** 与我们自研路线相反,不借鉴(同 JvedioNext)。
2. **演员维度是它的强项**,也正好戳中我们一直缺的环节:头像 + 资料。
3. **演员头像第四次出现**,且这次给了**具体可用的头像源 gfriends**(GitHub 仓库,按女优名取图)。结合 nassav.md 的「JavBus 详情页直接抓头像」,头像方案可双路:**JavBus 详情页头像为主、gfriends 兜底/补全**。
4. **男优刮削**是个少见但完整的维度补充,可作为可选项。
